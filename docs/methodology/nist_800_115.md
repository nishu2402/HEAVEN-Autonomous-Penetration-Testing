# HEAVEN ↔ NIST SP 800-115

Mapping of HEAVEN modules against [NIST SP 800-115 (Technical Guide to
Information Security Testing and Assessment)](https://csrc.nist.gov/publications/detail/sp/800-115/final).

NIST 800-115 is methodology-centric rather than test-list-centric, so this
mapping is per section rather than per test ID.

## Coverage summary

| Section | Coverage level |
|---|---|
| §3 Review Techniques | partial (config check in scanners) |
| §4 Target Identification & Analysis | extensive (recon + scanners) |
| §5 Target Vulnerability Validation | extensive (safe_validator, fp_suppress) |
| §6 Security Assessment Planning | manual + engagement DB |
| §7 Security Assessment Execution | extensive (orchestrator) |
| §8 Post-Testing Activities | extensive (reporting, evidence) |

## Audit-trail features (NIST 800-115 implicit requirements)

NIST emphasizes auditability throughout. HEAVEN supports this via:

- **Audit log**: every scan action logged to `data/audit/` (JSON-structured)
- **Evidence packaging**: `heaven/devsecops/evidence.py` produces curl-replayable repros
- **Engagement DB**: SQLite per engagement, contains full scan history, scope changes, finding status updates
- **Operator notes**: each finding has an `operator_notes` field, modifiable via `heaven mark <id> <status> --notes "..."`
- **Chain-of-custody**: timestamps on every event (`first_seen_at`, `last_seen_at`)
